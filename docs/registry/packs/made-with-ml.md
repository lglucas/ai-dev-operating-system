## What it is

A course teaching how to build production-grade machine-learning applications by combining ML with software-engineering principles. Covers the full lifecycle: data handling, model training, evaluation, serving, MLOps, distributed training, CI/CD, and LLM integration.

## When to consult

- Team transitioning from research ML to production ML.
- Before designing the MLOps layer of a project.
- Engineers and data scientists who need a shared production playbook.

## When NOT to consult

- Project only consumes pre-trained models via API.
- Team is purely research-focused.

## How to use

Reference course — no install. Run notebooks and follow the lifecycle.

## Fit signals

- TECHNICAL-PLAN.md includes ML model training, serving, or MLOps.
- Project will deploy and maintain custom ML models in production.

## Conflicts and overlaps

- Builds on top of `microsoft-ml-for-beginners` — that one teaches algorithms; this one teaches productionization.

## Notes

Distributed-training and PyTorch focus assumes GPU access for parts of the curriculum.
