# Tag: design

Packs in this category address UI/UX exploration, design systems, and visual prompt structuring for AI-generated interfaces.

| Pack | Focus |
|------|-------|
| [`ui-ux-pro-max`](../packs/ui-ux-pro-max.md) | Multi-direction UI exploration workflow |
| [`awesome-design-md`](../packs/awesome-design-md.md) | Design-system-as-prompt assets |

## When this tag is relevant

- Project includes user-facing UI surfaces (web, mobile, dashboard).
- WIZARD is at or past the Prototype Lab phase.
- Team needs structured UI decisions or consistent design enforcement.
- Project quality is measured partly by UI polish.

## How to choose between them

- Greenfield, no design opinions yet → `ui-ux-pro-max` (exploration).
- Existing brand or design system to enforce → `awesome-design-md` (consistency).
- Both can coexist — explore first, then enforce.
