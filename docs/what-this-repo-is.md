# What this repo is

AI Dev Operating System is a day-zero operating system for AI-assisted SaaS development.

It is not just a collection of prompts.

It is a reusable project structure that combines:

- rules
- agents
- skills
- commands
- sprint management
- changelog discipline
- session logs
- design workflows
- security gates
- attribution discipline

The goal is simple:

> A builder should be able to clone this repo and start a serious AI-assisted SaaS project without organizing the operating environment from scratch.

---

## What this repo is not

This repo is not:

- a SaaS boilerplate with finished app code
- a UI kit
- a specific Next.js/Supabase template
- a replacement for Claude Code, Cursor, Windsurf, or other IDEs
- a copy of upstream skill repos
- a dump of private project documentation
- a replacement for security review, legal review, or production hardening

It is the operating layer around AI-assisted development.

---

## Why it exists

AI coding agents are fast, but fast without structure creates chaos.

Common problems:

- The agent forgets project decisions.
- Every project starts from zero.
- Rules live in chat history instead of the repo.
- Features are built without sprint discipline.
- Changelog and rationale are missing.
- Security review becomes an afterthought.
- Design direction is improvised.
- External inspirations are used without proper attribution.

This repo tries to solve that by making the operating system explicit.

---

## The core loop

1. Define the project constitution in `CLAUDE.md`.
2. Add modular rules.
3. Choose agents.
4. Use skills for repeatable workflows.
5. Trigger work through commands.
6. Plan and close sprints.
7. Keep a changelog.
8. Log important decisions in session logs.
9. Run security and privacy checks.
10. Attribute sources and upstream inspirations.

---

## Best use case

This works best for:

- solo founder SaaS
- small teams
- fast MVP development
- agentic IDE workflows
- Next.js/Supabase apps
- product experiments
- web3 products with optional stack packs
- projects where continuity and decision memory matter
