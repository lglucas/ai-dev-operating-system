# Public Sanitization Notes

This repository is intended to remain public and reusable.

Before adding material from real projects, check whether the content contains:

- private business plans;
- customer data;
- supplier data;
- pricing strategy not intended for publication;
- private legal or compliance analysis;
- non-public product strategy;
- private GitHub links that should not be referenced;
- personal information beyond intentional author/maintainer attribution;
- credentials, secrets, tokens, API keys, or private URLs.

---

## What can be public

Generally safe:

- generic rules;
- generic agents;
- generic skills;
- command templates;
- installation docs;
- project operating-system philosophy;
- example wizard flows;
- placeholder examples;
- public links intentionally listed in `ATTRIBUTIONS.md` and `UPSTREAM-SOURCES.md`.

---

## What should stay private

Keep out of the public repo:

- real customer names;
- real contracts;
- private market intelligence;
- unpublished startup strategy;
- actual financial projections for private ventures;
- private API endpoints;
- private credentials;
- legal analysis tied to active private cases;
- internal communication logs.

---

## Public author attribution

The repository may intentionally include public maintainer attribution and public links selected by the maintainer.

Current public maintainer links are maintained in:

- `README.md`
- `ATTRIBUTIONS.md`
