# Decision Log

Use this when a decision affects architecture, product scope, pricing, compliance, security, or release strategy.

## Steps

1. State the decision.
2. Record context.
3. List alternatives considered.
4. Explain why the chosen path wins.
5. Note risks and follow-ups.
6. Link related sprint/changelog/issue.
