## Decisões

### 1. Separação `course/` como vertical paralela

Considerei 4 opções de organização:

| Opção | Vantagem | Desvantagem |
|---|---|---|
| A. Tudo na raiz, misturado | Nenhuma | Conflito visual com OS, polui o template. |
| B. Repo separado (`ai-dev-operating-system-course`) | Limpo | Sincronização entre 2 repos vira atrito. |
| C. Pasta `course/` no mesmo repo, com `.gitattributes export-ignore` | Limpo + 1 repo | "Use this template" não honra `export-ignore` (limitação GitHub). |
| D. Pasta `course/` no mesmo repo + sinalização forte ("não é parte do OS") | 1 repo, versionamento conjunto, sem conflito | Aumenta peso do template. |

**Escolha:** D. Custo de aumentar tamanho do template é menor que custo de manter 2 repos. Sinalização visual forte no `course/README.md` (banner ⚠️ + manifesto + comando `rm -rf course/` documentado).

### 2. Versionamento como v0.4.5 (patch), não v0.5.0

Considerei v0.5.0 (minor) por adicionar área inteira nova. Decisão pelo patch porque:

- **Nenhuma alteração no OS operacional** — WIZARD, regras, skills, registry, agents intocados.
- v0.5.0 já está reservado pras decisões arquiteturais do OS (trust tiers, plugin manifest, etc.) — anunciadas no v0.4.4.
- A vertical educacional é aditiva e isolada.

User concordou no enquadramento.
