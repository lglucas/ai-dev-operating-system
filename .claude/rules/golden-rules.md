# Golden Rules

1. Do not code before context.
2. Follow `START-HERE.md` and `WIZARD.md`.
3. Keep documentation layers separate.
4. Update changelog and session logs for material changes.
5. Use source-backed research.
6. Use red team review before finalizing BP and sprint plans.
7. Use stable technology by default.
8. Keep code modular and readable.
9. Use environment variables, never hardcoded secrets.
10. Prefer explicit assumptions over hidden guesses.
